## O que é Service Desk

Service Desk é uma área responsável por fornecer suporte aos usuários de uma organização, atuando como ponto central de contato entre usuários e a equipe de tecnologia.

O objetivo é registrar, analisar, solucionar e acompanhar solicitações e incidentes, garantindo a continuidade dos serviços de TI.

O Service Desk trabalha seguindo boas práticas de gerenciamento de serviços de TI (ITSM), buscando melhorar a qualidade do atendimento e reduzir impactos nos usuários.


---

# Diferença entre Help Desk e Service Desk

## Help Desk

O Help Desk possui foco principalmente na resolução de problemas técnicos dos usuários.

Exemplos:

- Problemas de senha
- Computador sem internet
- Instalação de programas
- Erros básicos de sistema


## Service Desk

O Service Desk possui uma atuação mais ampla.

Além de resolver problemas, ele também:

- Gerencia chamados
- Controla SLA
- Analisa impactos
- Registra soluções
- Melhora processos
- Atua como ponto único de contato da TI


---

# Analista N1

O analista N1 é o primeiro nível de atendimento.

Responsabilidades:

- Receber chamados
- Registrar informações
- Identificar o problema
- Realizar diagnósticos iniciais
- Resolver incidentes simples
- Orientar usuários
- Escalar chamados quando necessário


Exemplos de atividades:

- Reset de senha
- Configuração básica
- Problemas de rede simples
- Instalação de software


---

# Analista N2

O analista N2 possui conhecimento mais avançado e recebe chamados escalados pelo N1.

Responsabilidades:

- Investigar problemas mais complexos
- Analisar sistemas e infraestrutura
- Trabalhar com permissões
- Resolver problemas de servidores e redes


Exemplos:

- Problemas com Active Directory
- Falhas em serviços
- Configurações de rede avançadas


---

# Analista N3

O analista N3 representa o nível mais especializado.

Normalmente envolve equipes como:

- Administradores de sistemas
- Engenheiros de infraestrutura
- Desenvolvedores
- Especialistas de segurança


Responsabilidades:

- Resolver problemas críticos
- Analisar causa raiz
- Corrigir falhas complexas
- Criar melhorias permanentes


Exemplos:

- Falhas em aplicações
- Problemas de arquitetura
- Vulnerabilidades de segurança

# Ciclo de Vida de um Chamado

Um chamado normalmente segue as etapas:


## 1. Abertura

Usuário informa o problema ou solicita um serviço.


## 2. Registro

O chamado recebe:

- Identificação
- Usuário
- Data
- Categoria
- Descrição


## 3. Classificação

O chamado é categorizado.

Exemplos:

- Rede
- Hardware
- Software
- Segurança


## 4. Priorização

É definido o nível de prioridade considerando:

- Impacto
- Urgência


## 5. Diagnóstico

O analista realiza testes para encontrar a causa.


## 6. Resolução

O problema é corrigido ou uma solução é aplicada.


## 7. Encerramento

O chamado é finalizado com:

- Solução aplicada
- Registro das ações
- Feedback ao usuário
